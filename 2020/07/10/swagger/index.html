Swagger

公司最近开始了一个新项目，采用的前后端分离的开发方式。现在总结一下。

  • Swagger的作用和概念
  • 前后端分离
  • 在SpringBoot中集成Swagger

Swagger的作用和概念

前后端分离
VUE+SpringBoot

以前的开发是 后端时代:前端只用管理静态页面；html=>后端。模板引擎JSP=>后端是主力

前后端分离式时代：

  • ·后端︰后端控制层，服务层，数据访问层【后端团队】
  • ·前端︰前端控制层，视图层【前端团队】
  • 前后端交互是通过===》API
  • 前后端相对独立，松耦合
  • 前后端可以部署在不同服务器上

由此诞生出一个问题

  • 前后端集成联调，前后端人员对接不及时。

解决方案

  • 制定开发手册，更新最新API

  • 前后端分离测试：后端采用postman ，后端接口变动及时通知前端

Swagger

  • 号称世界上最流行的API框架
  • resultFul API 文档在线自动生成工具=》API文档与API定义同步更新
  • 直接运行，可在线测试接口
