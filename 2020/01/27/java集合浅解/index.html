一：什么是集合

在java中我们可以用数组来存储同类型的数据但是这样存储数据，数据的个数是固定的，类型也是确定的。也可以使用符串缓冲区，它可以保存任意类型的数据，最后它会把其中的所有数据全部转成字符串处理。随着类和接口的使用，我们创建的对象会愈来愈多，这些对象也需要一个空间进行存储，于是Java中针对对象这类数据提供集合这类容器保存。

集合就是一个用来保存对象的容器，集合在存储对象的时候可以根据对象的增减，而改变自己的容量，并且可以对存储进里面的数据进行操作。

二：集合框架

List接口实现了ArrayList类和LinkList类。

Set接口实现了HashSet类和LinkedHashSet类

boolean removeAll(Collection<?> c):删除交集数据

Boolean retainAll(Collection c):保留相同元素(交集)

2：迭代器

Collection集合元素的通用获取方式：在取元素之前先要判断集合中有没有元素，如果有，就把这个元素取出来，继续在判断，如果还有就再取出出来。一直把集合中的所有元素全部取出。这种取出方式专业术语称为迭代。集合中把这种取元素的方式描述在Iterator接口中。Iterator接口的常用方法如下

hasNext（）方法：判断集合中是否有元素可以迭代

next（）方法：用来返回迭代的下一个元素，并把指针向后移动一位。

6：LinkedList

List 接口的链接列表实现。实现所有可选的列表操作，并且允许所有元素（包括 null）。除了实现 List 接口外，LinkedList 类还为在列表的开头及结尾 get、remove 和 insert 元素提供了统一的命名方法。这些操作允许将链接列表用作堆栈、队列或双端队列。

LinkedList集合：它是List接口的一个直接实现类，它实现了List接口中的所有方法，LinkedList和ArrayList都是List接口的实现，但是ArrayList的底层使用的数组作为存储数据的结构，而LinkedList底层使用的链表作为数据的存储结构。

方法

LinkedList底层使用的是链表结构,因此增删快,查询相对ArrayList较慢

void addFirst(E e) :向链表的头部添加元素

void addLast(E e):向链表的尾部添加元素

E getFirst():获取链头的元素,不删除元素

E getLast():获取链尾的元素,不删除元素

E removeFirst():返回链头的元素并删除链头的元素

E removeLast():返回链尾的元素并删除链尾的元素

7：Vector集合

Vector集合是在JDK1.0版本的时候出现的一个可以保存元素的集合对象。

Vector它的底层和ArrayList一致，都采用的是可变数组。从JDK1.2之后，它被收编到集合框架中，变成了List接口下的一个实现类。后期只要使用Vector的地方都可以使用ArrayList代替。

Vector它的底层是线程同步（多个线程操作安全）。ArrayList多个线程操作不安全。

Itrator是JDK1.2之后出现的专门用于遍历集合的迭代器对象。在JDK1.0的时候，Vector集合使用需要Enumeration进行迭代。
